# V$LOGMNR_CONTENTS

V$LOGMNR_CONTENTS contains log history information. To query this view, you must have the LOGMINING privilege.

## Usage Notes

Previous Next JavaScript must be enabled to correctly display this content When a SELECT statement is executed against the V$LOGMNR_CONTENTS view, the archive redo log files are read sequentially. Translated records from the redo log files are returned as rows in the V$LOGMNR_CONTENTS view. This continues until either the filter criteria specified at startup ( EndTime or endScn ) are met or the end of the archive log file is reached. When this view is queried from a PDB, it returns only redo generated by that PDB.
