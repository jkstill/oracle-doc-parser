# DBMS_AUTO_SQLTUNE.EXECUTE_AUTO_TUNING_TASK

This function and procedure executes the Automatic SQL Tuning task ( SYS_AUTO_SQL_TUNING_TASK ).

## Syntax

```sql
DBMS_AUTO_SQLTUNE.EXECUTE_AUTO_TUNING_TASK(
   execution_name    IN VARCHAR2               := NULL,
   execution_params  IN DBMS_ADVISOR.argList   := NULL,
   execution_desc    IN VARCHAR2               := NULL)
 RETURN VARCHAR2;
 
DBMS_AUTO_SQLTUNE.EXECUTE_AUTO_TUNING_TASK(
   execution_name    IN VARCHAR2               := NULL,
   execution_params  IN DBMS_ADVISOR.argList   := NULL,
   execution_desc    IN VARCHAR2               := NULL);
```

## Parameters

| Parameter | Type | Mode | Description |
|-----------|------|------|-------------|
| execution_name | VARCHAR2 | IN | A name to qualify and identify an execution. If not specified, it is generated by the advisor and returned by function. |
| execution_params | DBMS_ADVISOR.argList | IN | List of parameters (name, value) for the specified execution. The execution parameters have effect only on the execution for which they are specified. They override the values for the parameters stored in the task (set through the SET_AUTO_TUNING_TASK_PARAMETER Procedures ). |
| execution_desc | VARCHAR2 | IN | A 256-length string describing the execution |

**Returns:** `VARCHAR2`

## Usage Notes

Both the function and the procedure run in the context of a new task execution. The difference is that the function returns the name of the new execution.
